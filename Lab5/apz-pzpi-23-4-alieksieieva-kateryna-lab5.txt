3	ОПИС ВИКОНАНОЇ РОБОТИ 
3.1 Загальний опис 
Reminder Power — це програмна система для автоматизації процесів створення, налаштування та керування підписками між бізнесами та їх клієнтами. Основною метою системи є спрощення роботи компаній із тарифними планами, автоматизація процесів оформлення підписок та забезпечення зручного контролю активних сервісів для кінцевих користувачів.
Система складається з трьох основних компонентів: веб-застосунку для користувачів, адміністративної панелі для бізнесів та адміністраторів, а також серверної частини, яка реалізує бізнес-логіку та взаємодію з базою даних PostgreSQL через REST API.
Серверна частина реалізована на платформі Spring Boot та підтримує локальне розгортання або запуск у контейнеризованому середовищі Docker і Kubernetes.
Система забезпечує керування тарифними планами, підключення користувачів до підписок, розрахунок вартості послуг з урахуванням додаткових компонентів, адміністрування бізнес-акаунтів, контроль життєвого циклу підписок та збір статистики використання сервісу.
3.2 Реалізовані функції 
Серверна частина реалізує механізми реєстрації та автентифікації користувачів на основі JWT-токенів, безпечне зберігання паролів із використанням BCrypt, а також рольову модель доступу для адміністраторів та звичайних користувачів.
Для бізнес-користувачів реалізовано можливість створення шаблонів підписок, налаштування терміну дії тарифних планів, базової вартості та додаткових компонентів послуг. Також реалізовано функціонал активації та деактивації тарифних планів без їх видалення із системи.
Для користувачів реалізовано перегляд доступних тарифних планів, оформлення підписки, автоматичний розрахунок кінцевої вартості послуги, перегляд активних підписок та можливість їх дострокового скасування.
Адміністративна панель забезпечує перевірку та модерацію бізнес-акаунтів, перегляд статистики системи, управління користувачами та контроль роботи сервісу.
Додатково реалізовано підтримку контейнеризації через Docker, розгортання в Kubernetes та можливість горизонтального масштабування серверної частини системи. 
3.3 Не реалізовані функції 
До подальших напрямків розвитку системи належать:
•	інтеграція з платіжними системами Stripe та PayPal для автоматичного проведення платежів;
•	автоматичне продовження підписок після завершення терміну дії;
•	надсилання email та push-нагадувань про завершення підписки;
•	інтеграція з календарями Google Calendar та Outlook для створення нагадувань;
•	рекомендаційна система тарифних планів на основі історії використання;
•	генерація фінансових звітів для бізнес-користувачів;
•	аналітика використання сервісів за допомогою технологій штучного інтелекту;
•	мобільний застосунок для iOS.

До переліку майбутніх інтеграцій та нереалізованого функціоналу входять: інтеграція з платіжним сервісом Stripe для проведення реальних онлайн-оплат; інтелектуальна система рекомендацій страв на основі попередніх замовлень; автоматичне прокладання маршрутів доставки з інтеграцією інтерактивних карт; ШІ-аналітика для прогнозування попиту залежно від сезону чи дня тижня; мультимедійний чат у реальному часі між клієнтом та персоналом ресторану; повна інтеграція з кухонними IoT-дисплеями для автоматичного трекінгу етапів приготування без ручного введення. 
4	ВИСНОВКИ 
Під час виконання курсового проєкту було розроблено інформаційну систему Reminder Power для керування підписками між бізнесами та користувачами.
У процесі розробки було реалізовано багаторівневу архітектуру застосунку на основі Spring Boot, PostgreSQL та REST API. Для забезпечення безпеки системи використано JWT-аутентифікацію, рольову модель доступу та шифрування паролів за допомогою BCrypt.
Було створено модулі реєстрації та авторизації користувачів, управління бізнес-акаунтами, створення та налаштування шаблонів підписок, оформлення користувацьких підписок та адміністрування системи. Реалізовано механізм розрахунку кінцевої вартості підписки з урахуванням додаткових компонентів сервісу.
Особливу увагу приділено можливості масштабування системи. Для цього підготовлено Docker-конфігурації та Kubernetes-манифести, що дозволяють виконувати розгортання застосунку в контейнеризованому середовищі та забезпечують готовність до роботи під навантаженням.
У результаті виконання роботи було створено працездатний програмний продукт, який відповідає поставленим функціональним вимогам та може бути використаний як основа для подальшого розвитку комерційної платформи управління підписками.
